一、为AI深度学习准备能进行的环境配置——默认已有python3环境

Anaconda——python的包管理和环境管理器

Q：为什么要安这个？（觉得不需要可以不安）
A：方便配置py的虚拟编译环境，降低python各种库的试错成本和学习尝试风险
Q：怎么安？
A：[官网](Anaconda | Anaconda Distribution)下载安装（太慢啦）或国内镜像软件源下载安装（快）例如[清华的镜像站](Index of /anaconda/archive/ | 清华大学开源软件镜像站 | Tsinghua Open Source Mirror)
注：
  • 官网下载点击之后可能会跳转订阅注册页面，此时需要鼠标右击下载按钮选择在新标签页中打开就可以直接下载了
  • 在镜像站下载要注意版本问题，一般根据不同项的名称和上传时间确认
  • 本人选择的是Anaconda3-2022.10-Windows-x86_64.exe (可执行安装程序)
  • 安装时最好勾选上自动把anaconda添加进PATH（系统环境变量）里
  • 安装完成后在cmd里输入conda list 检查安装是否成功
另外，安装好anaconda后，我们可以搭建一个简单的py虚拟编译环境

  1. 先对anaconda进行换源，具体详见anaconda | 镜像站使用帮助 | 清华大学开源软件镜像站 | Tsinghua Open Source Mirror

  2. 选择一个位置（文件夹）放置新建的环境，在命令行中cd 进入该文件夹（在Anaconda文件夹里会直接放到Anaconda\envs\路径下）

  3. 使用conda create -n 环境名 python=版本号 -y新建一个环境，例如:使用

    conda create -n pytorch_env python=3.9 ,遇到y/n 输入y即可

  4. 创建完成后，输入命令conda env list，显示现有的环境（前面是新建时取的环境名字，后面是环境所在文件路径，base是默认自带的）：

Pytorch

获取安装命令

这个玩意通过命令行安装,，具体命令可以百度看博客，也可以直接到[官网](Start Locally | PyTorch)根据个人使用平台查看最新安装命令（到官网后，进入”Get Started“下，下拉到如下图的page页面并根据个人平台选择）,

  1. 上图第一行”PyTorch Build“的三个选项分别是Stable（稳定版）、Preview（先行测试版）、LTS（长期支持版），选择第一个和第三个都无所谓，但不推荐选第二个
  2. 上图第二行”Your OS“选择本机操作系统就行
  3. 上图第三行”Package“，选择用那个包管理工具（平台）进行安装，分别是Conda（Anaconda命令行包安装工具）、Pip（Python自带的包安装工具）、LibTorch（不知道，应该和Torch本身的安装工具有关）、Source（直接在github上下载pytorch的代码，然后自己setup，小白别用）
  4. 上图第四行”Language“，选python，毕竟除了某些特殊群体，一般没谁会傻到用Java跑AI深度学习吧···
  5. 上图第五行”Computer Platform“，选择用什么编程算法软件跑，前两个CUDA项是皮衣黄家的显卡可以使用的平台软件版本（Windows、Linux、Mac都可以下载到CUDA，但老黄对Linux的驱动支持那可真是一绝啊~滑稽），第三个项ROCm是AMD为了”对标“CUDA出的一个机器学习库，但由于各种原因，只能在Linux上跑，而且问题比较多，支持的型号比较少（有得用就不错了），最后一个CPU项，顾名思义，就是给我这样的倒霉蛋用的让cpu来跑机器学习的选择，比较欣慰的是，前三项要匹配库与pytorch的版本，但CPU项不需要（还算一点安慰）
  6. 上图最后一行，根据前面的做出选择给出的最新下载命令，直接cv到 上面选择的Package对应的工具里下载

    注：CPU算力珍贵，且用且珍惜

    • Prompt：这一行设置希望AI生成的画作中包含的“元素”，用英文词汇描述，例如可以输入：

      当然，毕竟CPU算图得慢，所以核显佬不要随便就找些关键词扔上去，不然花了时间也没得到想要的图，心里也不爽···

      一些可用的关键词记录

    • Negative prompt ：和上面相反，设置不希望在AI生成的画作中包含的“元素”，这个一般是通用的（毕竟人的xp可能千奇百怪，但列举大部分人都不喜欢的东西还是很容易的，另外，记得在这里防范下奇奇怪怪的克苏鲁元素，防止精神污染，比如：“奇怪的触手”之类的）

    • Sampling Steps（采样步长）：简单地理解为迭代次数，越大越耗时，一般20左右就足够了，太大提升也有限，（有细致需求30也行，不过核显是不要想了···太大简直跑到想爆炸）。

    • Sampling method（采样方法）：简单地理解为迭代算法，感觉Euler和DDIM（Novel特色）就不错，不同算法适合的迭代次数不尽相同。

      （注：DDIM（Novel特色）只能用Nvidia显卡的cuda来跑，核显可以选择Euler或Euler a）

    • Width、Height：控制图像大小，设备性能差就不要妄想开太大，而且这个项目也不适合生成太大的···（512就挺好）

    • Batch count（连续生成次数）、Batch size（每次生成张数）：控制每次任务的出图数量，Batch size太大爆显存，CPU算的话似乎用的是内存（核显佬唯一有优势的地方···但8g以下不要想着多生成了···）

    • CFG Scale（参数制约度）：简单地讲，AI的听话程度，越大越听话（遵照Prompt、Negative prompt），越小越“放荡不羁爱自由”

    • Seed：就是随机种子，可以指定，-1就是随机种子。
